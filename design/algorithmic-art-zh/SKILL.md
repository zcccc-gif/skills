# 算法艺术技能

算法哲学是计算美学运动，随后通过代码表达。输出 .md 文件（哲学）、.html 文件（交互查看器）和 .js 文件（生成算法）。

分两步完成：
1. 算法哲学创建（.md 文件）
2. 通过 p5.js 生成艺术表达（.html + .js 文件）

## 算法哲学创建

首先创建一个**算法哲学**（非静态图像或模板），通过以下方式诠释：
- 计算过程、涌现行为、数学之美
- 种子随机性、噪点场、有机系统
- 粒子、流、场、力
- 参数变化和受控混沌

### 关键理解
- **收到什么**：用户的微妙输入或指示，作为基础而非约束
- **创建什么**：算法哲学/生成美学运动
- **接下来**：同一版本接收哲学并**用代码表达** - 创建 90% 算法生成、10% 必要参数的作品

### 如何生成算法哲学

**命名运动**（1-2 个词）：
- "有机湍流" / "量子谐波" / "涌现寂静" / "场域动力" / "随机结晶"

**阐述哲学**（4-6 段 - 简洁但完整）：

表达哲学如何通过以下方式体现：
- 计算过程和数学关系？
- 噪点函数和随机模式？
- 粒子行为和场域动态？
- 时间演化和系统状态？
- 参数变化和涌现复杂性？

**关键指南：**
- **避免冗余**：每个算法方面只提及一次
- **反复强调工艺**：最终算法应看起来花费了无数小时开发
- **留出创意空间**：具体说明算法方向，但保持简洁

### 哲学示例

**"有机湍流"**
哲学：混沌受自然法则约束，秩序从无序中涌现。
算法表达：由多层 Perlin 噪点驱动的流场。成千上万的粒子跟随矢量力，它们的轨迹累积成有机密度图。颜色从速度和密度中涌现。

**"量子谐波"**
哲学：离散实体表现出类似波的干涉图案。
算法表达：粒子在网格上初始化，每个携带通过正弦波演化的相位值。当粒子靠近时，它们的相位干涉 - 相干干涉产生明亮节点，破坏性干涉产生空洞。

**"递归低语"**
哲学：跨尺度的自相似性，有限空间中的无限深度。
算法表达：递归细分的分支结构。每个分支略微随机化但受黄金比例约束。L 系统或递归细分生成既数学又有机的树状形式。

### 基本原则
- **算法哲学**：创建通过代码表达的计算世界观
- **过程优于产品**：强调美来自算法的执行 - 每次运行都是独特的
- **参数表达**：思想通过数学关系、力、行为交流
- **艺术自由**：下一个 Claude 算法诠释哲学 - 提供创意实现空间
- **纯粹生成艺术**：这是制作活的算法，不是带随机性的静态图像

**算法哲学应为 4-6 段长。** 输出为 .md 文件。

## P5.JS 实现

哲学和概念框架确立后，用代码表达。

### 技术要求

**种子随机性**：
```javascript
// 始终使用种子以确保可复现性
let seed = 12345;
randomSeed(seed);
noiseSeed(seed);
```

**参数结构**：
```javascript
let params = {
  seed: 12345,  // 始终包含种子
  // 添加控制算法的参数：
  // - 数量（多少？）
  // - 尺度（多大？多快？）
  // - 概率（多可能？）
  // - 比例（什么比例？）
  // - 阈值（行为何时改变？）
};
```

**核心算法 - 表达哲学**：

**关键**：算法哲学应该决定构建什么。

如果哲学关于**有机涌现**，考虑：
- 随时间累积或增长的元素
- 受自然规则约束的随机过程
- 反馈循环和交互

如果哲学关于**数学之美**，考虑：
- 几何关系和比例
- 三角函数和谐波
- 精确计算创造意外图案

### 工艺要求

**关键**：要达到大师级，创建感觉像经过无数次迭代的算法。仔细调整每个参数。

- **平衡**：复杂而不杂乱，有序而不僵化
- **色彩和谐**：深思熟虑的调色板
- **构图**：即使在随机中，保持视觉层次和流动
- **性能**：流畅执行
- **可复现性**：相同的种子始终产生相同的输出

### 输出格式

输出：
1. **算法哲学** - 解释生成美学的 markdown
2. **单个 HTML 作品** - 自包含的交互式生成艺术

HTML 作品包含一切：p5.js（CDN）、算法、参数控件和 UI - 全部在一个文件中。

## 交互式作品创建

### 必需功能

**1. 参数控件**
- 数字参数的滑块
- 调色板的颜色选择器
- 参数更改时实时更新
- 重置按钮恢复默认值

**2. 种子导航**
- 显示当前种子编号
- "上一个"和"下一个"按钮循环种子
- "随机"按钮获取随机种子
- 输入框跳转到特定种子

**3. 单个作品结构**
```html
<!DOCTYPE html>
<html>
<head>
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 所有样式内联 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 所有参数控件 -->
  </div>
  <script>
    // 所有 p5.js 代码内联
  </script>
</body>
</html>
```

**关键**：这是单个作品。无外部文件，无导入（除 p5.js CDN）。一切内联。

## 创意过程

**用户请求** → **算法哲学** → **实现**

每个请求都是独特的。过程包括：

1. **诠释用户意图** - 寻求什么美学？
2. **创建算法哲学**（4-6 段）描述计算方法
3. **用代码实现** - 构建表达此哲学的算法
4. **设计适当的参数** - 什么应该是可调的？
5. **构建匹配的 UI 控件** - 参数的滑块/输入
