# 前端设计指南

本技能指导创建独特、生产级的前端界面，避免通用的"AI 味"美学。实现真正可用的代码，注重美学细节和创意选择。

## 设计思维

在编码之前，理解上下文并选择一个**大胆**的美学方向：
- **目的**：这个界面解决什么问题？谁使用它？
- **基调**：选择一个极端方向：极简主义、极繁主义、复古未来、有机/自然、奢华/精致、活泼/玩具感、杂志/编辑、粗野主义、装饰艺术/几何、柔和/粉彩、工业/实用等
- **约束**：技术要求（框架、性能、无障碍）
- **差异化**：什么让这个设计**令人难忘**？

**关键**：选择清晰的概念方向并精确执行。大胆的极繁主义和精致的极简主义都有效 - 关键是有意识，而非强度。

## 前端美学指南

### 字体
- 选择美丽、独特、有趣的字体
- 避免通用字体如 Arial、微软雅黑（正文可用，标题需独特）
- 将独特的展示字体与精致的正文字体配对
- **中文字体推荐**：
  - 标题：思源黑体、华文黑体、方正兰亭黑
  - 正文：微软雅黑、苹方、思源宋体

### 颜色与主题
- 坚持统一的美学
- 使用 CSS 变量保持一致性
- 主导色配鲜明强调色优于平庸的均匀分布

### 动效
- 使用动画效果和微交互
- HTML 优先使用纯 CSS 方案
- React 可使用 Motion 库
- 关注高影响力时刻：一个精心编排的页面加载动画比分散的微交互更有愉悦感

### 空间构图
- 意外的布局、不对称、重叠、对角线流动
- 打破网格的元素
- 充裕的负空间或受控的密度

### 背景与视觉细节
- 创建氛围和深度，而非默认纯色
- 添加与整体美学匹配的上下文效果和纹理
- 应用渐变网格、噪点纹理、几何图案、分层透明度、戏剧性阴影等

## 绝对避免

- 过度使用的字体家族（Arial、系统字体）
- 陈词滥调的配色方案（特别是紫色渐变配白色背景）
- 可预测的布局和组件模式
- 缺乏上下文特征的千篇一律设计

## 实现原则

- 生产级且功能完整
- 视觉冲击力强且令人难忘
- 与清晰的美学观点一致
- 每个细节都精心打磨

**重要**：实现复杂度与美学愿景匹配。极繁主义设计需要精心编排的代码和大量动画。极简主义设计需要克制、精确和对间距、字体、细节的关注。
