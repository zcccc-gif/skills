## 设计理念

**不要创建无聊的幻灯片。** 白色背景上的普通项目符号不会给任何人留下深刻印象。

### 开始之前

- **选择大胆、内容相关的配色方案**：配色应该为主题量身定制
- **主次分明**：一种颜色应占主导地位（60-70%），1-2 种辅助色和一种强调色
- **明暗对比**：标题和结论幻灯片使用深色背景，内容页使用浅色（三明治结构）
- **统一视觉元素**：选择一个独特的元素并重复使用 — 圆形图片框架、图标在彩色圆圈中、粗单边框

### 配色方案

选择与主题匹配的颜色 — 不要默认使用蓝色。以下是一些参考：

| 主题 | 主色 | 辅助色 | 强调色 |
|-------|---------|-----------|--------|
| **商务蓝** | `1E2761`（深蓝） | `CADCFC`（冰蓝） | `FFFFFF`（白色） |
| **自然绿** | `2C5F2D`（森林） | `97BC62`（苔藓） | `F5F5F5`（米色） |
| **活力橙** | `F96167`（珊瑚） | `F9E795`（金色） | `2F3C7E`（深蓝） |
| **温暖棕** | `B85042`（赤陶） | `E7E8D1`（沙色） | `A7BEAE`（鼠尾草） |
| **海洋蓝** | `065A82`（深蓝） | `1C7293`（青色） | `21295C`（午夜） |
| **简约灰** | `36454F`（炭灰） | `F2F2F2`（米白） | `212121`（黑色） |
| **信赖青** | `028090`（青色） | `00A896`（海泡石） | `02C39A`（薄荷） |
| **优雅紫** | `6D2E46`（浆果） | `A26769`（玫瑰） | `ECE2D0`（奶油） |

### 每张幻灯片

**每张幻灯片都需要视觉元素** — 图片、图表、图标或形状。纯文字幻灯片容易被遗忘。

**布局选项：**
- 双栏（文字在左，图示在右）
- 图标 + 文字行（图标在彩色圆圈中，粗体标题，描述在下方）
- 2×2 或 2×3 网格
- 半出血图片（左侧或右侧满幅）配内容覆盖

**数据显示：**
- 大数字突出显示（60-72pt 数字配小标签）
- 对比栏（前后对比、优缺点、并排选项）
- 时间线或流程图（编号步骤、箭头）

### 字体

**中文字体推荐：**

| 标题字体 | 正文字体 |
|-------------|-----------|
| 微软雅黑 | 微软雅黑 |
| 思源黑体 | 思源黑体 |
| 苹方 | 苹方 |
| 华文黑体 | 华文宋体 |
| 方正兰亭黑 | 方正兰亭黑 |

| 元素 | 大小 |
|---------|------|
| 幻灯片标题 | 36-44pt 粗体 |
| 章节标题 | 20-24pt 粗体 |
| 正文 | 14-16pt |
| 注释 | 10-12pt 浅色 |

### 间距

- 最小边距 0.5 英寸
- 内容块之间 0.3-0.5 英寸
- 留出呼吸空间 — 不要填满每一寸

### 常见错误

- **不要重复相同的布局** — 在幻灯片之间变化栏、卡片和突出显示
- **不要居中正文** — 左对齐段落和列表；仅标题居中
- **不要忽视大小对比** — 标题需要 36pt+ 以区别于 14-16pt 正文
- **不要默认使用蓝色** — 选择反映特定主题的颜色
- **不要随意混合间距** — 选择 0.3" 或 0.5" 间隔并保持一致
- **不要只美化一张幻灯片** — 要么全部投入，要么保持简单
- **不要创建纯文字幻灯片** — 添加图片、图标、图表或视觉元素
- **不要在标题下使用装饰线** — 这是 AI 生成幻灯片的典型特征

## 质量检查（必须）

**假设有问题。你的工作是找到它们。**

### 内容检查

```bash
python -m markitdown output.pptx
```

检查缺失内容、错别字、顺序错误。

**使用模板时，检查遗留的占位符文本：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|测试"
```

### 视觉检查

**⚠️ 使用子代理** — 即使只有 2-3 张幻灯片。你会看到你期望看到的，而不是实际存在的。

将幻灯片转换为图片（参见[转换为图片](#转换为图片)），然后使用以下提示：

```
视觉检查这些幻灯片。假设有问题 — 找出它们。

检查：
- 元素重叠（文字穿过形状、线条穿过文字）
- 文字溢出或在边缘/框边界被切断
- 来源引用或页脚与上方内容冲突
- 元素太近（< 0.3" 间隔）
- 间距不均匀（一个地方大面积空白，另一个地方拥挤）
- 边距不足（< 0.5"）
- 列或类似元素未对齐
- 低对比度文字
- 文字框过窄导致过多换行

报告所有发现的问题，包括次要问题。
```

### 验证循环

1. 生成幻灯片 → 转换为图片 → 检查
2. **列出发现的问题**（如果没有找到，更批判性地再看一次）
3. 修复问题
4. **重新验证受影响的幻灯片** — 一个修复常常会引发另一个问题
5. 重复直到完整检查没有发现新问题

**在完成至少一次修复-验证循环之前，不要宣布成功。**

## 转换为图片

将演示文稿转换为单独的幻灯片图片进行视觉检查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

这会创建 `slide-01.jpg`、`slide-02.jpg` 等。
