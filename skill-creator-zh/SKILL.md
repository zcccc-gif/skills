# 技能创建器

用于创建新技能和迭代改进的技能。

## 核心流程

创建技能的高级流程：

1. 确定技能要做什么以及大致如何实现
2. 编写技能草稿
3. 创建测试提示并运行测试
4. 帮助用户定性和定量评估结果
5. 根据反馈重写技能
6. 重复直到满意
7. 扩大测试集并再次尝试

## 与用户沟通

注意上下文线索来理解如何措辞沟通。如果不确定用户是否理解术语，可以简要解释。

## 创建技能

### 捕获意图

首先理解用户意图。当前对话可能已包含用户想要捕获的工作流程。

1. 这个技能应该让 Claude 能做什么？
2. 什么时候应该触发？（用户什么话/什么场景）
3. 期望的输出格式是什么？
4. 是否需要设置测试用例来验证技能？

### 访谈和研究

主动询问边缘情况、输入/输出格式、示例文件、成功标准和依赖项。

### 编写 SKILL.md

基于用户访谈，填写以下组件：

- **name**：技能标识符
- **description**：何时触发、做什么。这是主要触发机制 - 包括技能做什么 AND 何时使用的具体上下文。所有"何时使用"信息放在这里，而非正文中。
- **compatibility**：所需工具、依赖项（可选，很少需要）
- **技能正文**

### 技能结构

```
skill-name/
├── SKILL.md（必需）
│   ├── YAML 前置元数据（name, description 必需）
│   └── Markdown 指令
└── 捆绑资源（可选）
    ├── scripts/    - 确定性/重复任务的可执行代码
    ├── references/ - 按需加载到上下文的文档
    └── assets/     - 输出中使用的文件（模板、图标、字体）
```

### 渐进式披露

技能使用三级加载系统：
1. **元数据**（name + description）- 始终在上下文中（~100 词）
2. **SKILL.md 正文** - 技能触发时在上下文中（理想 <500 行）
3. **捆绑资源** - 按需（无限制，脚本可以不加载执行）

**关键模式：**
- 保持 SKILL.md 在 500 行以内
- 从 SKILL.md 清晰引用文件
- 大型参考文件（>300 行）包含目录

### 编写模式

优先使用祈使形式。

**定义输出格式**：
```markdown
## 报告结构
始终使用此确切模板：
# [标题]
## 执行摘要
## 主要发现
## 建议
```

**示例模式**：
```markdown
## 提交信息格式
**示例 1：**
输入：添加了 JWT 令牌的用户认证
输出：feat(auth): 实现基于 JWT 的认证
```

### 测试用例

编写技能草稿后，想出 2-3 个现实的测试提示。

保存测试用例到 `evals/evals.json`：
```json
{
  "skill_name": "example-skill",
  "evals": [
    {
      "id": 1,
      "prompt": "用户的任务提示",
      "expected_output": "期望结果描述",
      "files": []
    }
  ]
}
```

## 运行和评估测试用例

### 步骤 1：同时生成所有运行

为每个测试用例，同时生成两个子代理 - 一个有技能，一个没有。

### 步骤 2：运行进行时，起草断言

不要等待运行完成。起草定量断言并向用户解释。

### 步骤 3：运行完成时，捕获计时数据

保存 `timing.json`：
```json
{
  "total_tokens": 84852,
  "duration_ms": 23332,
  "total_duration_seconds": 23.3
}
```

### 步骤 4：评分、聚合并启动查看器

1. **评分每个运行**
2. **聚合成基准**：`python -m scripts.aggregate_benchmark <workspace>/iteration-N --skill-name <name>`
3. **分析 pass**
4. **启动查看器**

### 步骤 5：读取反馈

用户完成后，读取 `feedback.json`。

## 改进技能

### 如何思考改进

1. **从反馈中泛化** - 创建可用于数百万次不同提示的技能
2. **保持精简** - 移除不起作用的内容
3. **解释原因** - 尝试解释一切的**原因**
4. **寻找重复工作** - 如果所有测试用例都写了类似的辅助脚本，捆绑它

### 迭代循环

改进技能后：
1. 应用改进
2. 重新运行所有测试用例
3. 启动查看器
4. 等待用户审查
5. 读取新反馈，再次改进，重复

## 描述优化

SKILL.md 中的 description 字段是决定 Claude 是否调用技能的主要机制。

### 步骤 1：生成触发评估查询

创建 20 个评估查询 - 应触发和不应触发的混合。

### 步骤 2：与用户审查

使用 HTML 模板展示评估集。

### 步骤 3：运行优化循环

```bash
python -m scripts.run_loop \
  --eval-set <path-to-trigger-eval.json> \
  --skill-path <path-to-skill> \
  --model <model-id> \
  --max-iterations 5 \
  --verbose
```

### 步骤 4：应用结果

从 JSON 输出中取 `best_description` 并更新技能的 SKILL.md。
